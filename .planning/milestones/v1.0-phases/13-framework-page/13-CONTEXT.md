# Phase 13: Framework Page - Context

**Gathered:** 2026-03-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Dedicated `/framework` route showing the full 30-step validation framework grouped by 4 phases, linked from the navbar (logged-in users only) and from the home page hero (always visible text link). The route itself is auth-gated.

</domain>

<decisions>
## Implementation Decisions

### Page layout
- **D-01:** Single column, full-width cards (max-w-2xl, matching HomePage/HistoryPage width)
- **D-02:** 4 `Card` components — one per framework phase — with steps listed inside each
- **D-03:** Cards stacked vertically (no 2-col grid), same pattern as existing pages
- **D-04:** Hero/intro section above the cards: heading ("The 30-Step Framework") + 1–2 line description explaining what the framework is for
- **D-05:** Step numbers styled large (e.g. `text-2xl font-heading`) — visually prominent anchor for each step
- **D-06:** No phase weighting shown (no "30%", "25%" labels on cards)

### Card decorations
- **D-07:** Phase 1 card → `decoration="tape"`, Phase 2 → `decoration="tack"`, Phase 3 → `decoration="tape"`, Phase 4 → `decoration="tack"` — alternating for variety, matching existing card usage

### Content
- **D-08:** Each step shows: large step number + step name + 1-sentence explanation of WHY it matters (strategic rationale, not tactical how-to)
- **D-09:** Framework data defined in a new `client/src/utils/frameworkSteps.js` file — data array, NOT imported from server/systemPrompt.js
- **D-10:** All 1-sentence explanations generated by the planner agent during planning (not placeholder TODOs)
- **D-11:** No scoring rules section on this page (no weights, no verdict thresholds)

### Navbar link
- **D-12:** Framework link appears in the navbar **only for logged-in users**, positioned between the Framework and History links: `Framework | History | Sign Out`
- **D-13:** Implemented as a `<Link to="/framework">` following the same pattern as the existing History link

### Home page hero link
- **D-14:** A text link appears below the description paragraph in the HomePage hero: e.g. "See the full 30-step framework →"
- **D-15:** Styled as `font-body text-blue hover:text-accent transition-colors` — same link styling as History in navbar

### Route & auth guard
- **D-16:** `/framework` route added to `App.jsx` `<Routes>` block
- **D-17:** Route is auth-gated: if user is not authenticated, redirect to `/` and open the auth modal (same pattern used by history-gated actions)
- **D-18:** The home hero text link is always visible regardless of auth state (public CTA that prompts login)

### Claude's Discretion
- Exact intro paragraph text for the hero section
- Precise wobbly border-radius variant per card (use existing `Card` component props)
- Rotation values for cards (use existing `rotate` prop)
- Spacing between steps within each card

</decisions>
