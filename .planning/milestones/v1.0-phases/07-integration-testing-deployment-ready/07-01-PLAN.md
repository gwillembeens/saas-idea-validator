# Phase 7 Plan 01: Error Display & Requirements Traceability

## Objective

Two gaps exist before the app is ready for end-to-end use:

1. **Silent error state**: When `status === 'error'`, `ResultsPanel` returns `null`. The user sees nothing — no message, no indication that something went wrong. The Redux slice stores the error message in `state.validator.error`, but it's never shown.

2. **Stale REQUIREMENTS.md**: Phases 4–6 completed COMP-01 through COMP-04 and DESIGN-01 through DESIGN-07, but the traceability table still shows these as "Pending". This is a documentation gap.

## Context

**Current ResultsPanel behavior:**
- `status === 'idle'` → returns null ✓
- `status === 'loading'` → shows skeleton ✓
- `status === 'streaming'` → falls through to markdown render ✓
- `status === 'done'` → shows markdown render ✓
- `status === 'error'` → returns null ✗ (user sees nothing)

**Error flow:**
- `useValidate` catches any fetch/stream error and dispatches `setError(e.message)`
- `setError` sets `status: 'error'` and `error: e.message` in the slice
- The `error` string is "Server error" (non-ok response) or the native Error message
- App.jsx renders results section when `status !== 'idle'` — so error state IS within the results container, but ResultsPanel bails early with null

**Design system rules:**
- Error/accent color: `#ff4d4d` (accent)
- All cards: use `Card` component with wobbly border
- Font: `font-body` for body text, `font-heading` for titles
- Hard shadow on Card

To:
```jsx
if (status === 'error') {

    <Card decoration="none" rotate={0} className="w-full max-w-2xl mx-auto">
      <div className="flex flex-col items-center gap-3 text-center py-4">
        <span className="text-accent text-4xl" aria-hidden="true">✕</span>
        <h3 className="font-heading text-2xl text-pencil">Validation Failed</h3>

      </div>
    </Card>
  )
}
```


To:
```jsx
const { status, result, error } = useSelector(s => s.validator)
```
