**Goal:** Create `likes` and `comments` tables with proper constraints and indexes

    ```sql
    CREATE TABLE IF NOT EXISTS likes (
      id SERIAL PRIMARY KEY,
      user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
      result_id INTEGER NOT NULL REFERENCES saved_results(id) ON DELETE CASCADE,

    ```

    ```sql
    CREATE TABLE IF NOT EXISTS comments (
      id SERIAL PRIMARY KEY,
      user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
      result_id INTEGER NOT NULL REFERENCES saved_results(id) ON DELETE CASCADE,
      parent_id INTEGER REFERENCES comments(id) ON DELETE CASCADE,
      body TEXT NOT NULL CHECK (char_length(body) <= 500),
      created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
      deleted_at TIMESTAMPTZ DEFAULT NULL
    );
    CREATE INDEX IF NOT EXISTS comments_result_id_idx ON comments(result_id);
    CREATE INDEX IF NOT EXISTS comments_parent_id_idx ON comments(parent_id);
    CREATE INDEX IF NOT EXISTS comments_user_created_idx ON comments(user_id, created_at);
    ```

### Task 23-01-03: Implement Comment Routes
**Goal:** Post/get/delete comment endpoints with threading and 500-char validation

**Files:**
- `server/routes/social.js` â€” Implement:
  - **`getCommentsRoute`** handles `GET /api/results/:id/comments` (public):
    - Validate result exists (404 if not found)
    - Query root comments with author username:
      ```sql
      SELECT c.id, c.user_id, u.username AS author_username, c.body, c.created_at
      FROM comments c LEFT JOIN users u ON c.user_id = u.id
      WHERE c.result_id=$1 AND c.parent_id IS NULL AND c.deleted_at IS NULL
      ORDER BY c.created_at ASC
      ```
    - For each root comment, query replies the same way with `parent_id=c.id`
    - Set `is_own: comment.user_id === req.user?.id` (or null if not logged in)
    - Return nested array: `[ { id, author_username, body, created_at, is_own, replies: [...] }, ... ]`
  - **`postCommentRoute`** handles `POST /api/results/:id/comments` (requires auth):
    - Validate result exists and is_public=true
    - Validate body: 1 â‰¤ length â‰¤ 500; return 400 if invalid
    - Insert: `INSERT INTO comments (user_id, result_id, parent_id, body) VALUES ($1,$2,NULL,$3) RETURNING *`
    - Return created comment with is_own=true
  - **`postReplyRoute`** handles `POST /api/comments/:id/replies` (requires auth):
    - Validate parent comment exists: `SELECT parent_id, result_id FROM comments WHERE id=$1 AND deleted_at IS NULL`
    - Validate parent_id IS NULL (no replies-to-replies); return 400 if parent is itself a reply
    - Validate parent result is_public=true
    - Validate body: 1 â‰¤ length â‰¤ 500
    - Insert with parent's result_id: `INSERT INTO comments (user_id, result_id, parent_id, body) VALUES ($1,$2,$3,$4) RETURNING *`
    - Return created reply with is_own=true
  - **`deleteCommentRoute`** handles `DELETE /api/comments/:id` (requires auth):
    - Validate comment exists: `SELECT user_id FROM comments WHERE id=$1`
    - Validate ownership: 403 if comment.user_id !== req.user.id
    - Soft delete: `UPDATE comments SET deleted_at=NOW() WHERE id=$1`
    - Return 204 No Content

### Task 23-01-04: Update Profile Route â€” Social Counts & UNION Heatmap
**Goal:** Extend `profileRoute` with like/comment counts per validation, total_likes_received, and unified heatmap

**Files:**
- `server/routes/profile.js` â€” Modify `profileRoute`:
  - **After fetching validations**, add a single batch query for like and comment counts:
    ```sql
    SELECT result_id,
      (SELECT COUNT(*)::int FROM likes l WHERE l.result_id = sr_counts.result_id) AS like_count,
      (SELECT COUNT(*)::int FROM comments c WHERE c.result_id = sr_counts.result_id AND c.deleted_at IS NULL) AS comment_count
    FROM (SELECT UNNEST($1::int[]) AS result_id) sr_counts
    ```
    Or simpler: query both in one pass for each validation.
  - **Update `formattedValidations` map** to include `like_count` and `comment_count` fields
  - **Add `total_likes_received` to stats**:
    ```sql
    SELECT COUNT(*)::int AS total
    FROM likes l
    INNER JOIN saved_results sr ON l.result_id = sr.id
    WHERE sr.user_id = $1 AND sr.is_public = true AND sr.deleted_at IS NULL
    ```
    Add to `stats` in response: `total_likes_received: number | null`
  - **Expand heatmap query to UNION all 3 activity types**:
    ```sql
    SELECT DATE(created_at) AS day, COUNT(*)::int AS count
    FROM (
      SELECT created_at FROM saved_results
        WHERE user_id=$1 AND is_public=true AND deleted_at IS NULL
        AND DATE(created_at)>=$2 AND DATE(created_at)<=$3
      UNION ALL
      SELECT created_at FROM likes
        WHERE user_id=$1
        AND DATE(created_at)>=$2 AND DATE(created_at)<=$3
      UNION ALL
      SELECT created_at FROM comments
        WHERE user_id=$1 AND deleted_at IS NULL
        AND DATE(created_at)>=$2 AND DATE(created_at)<=$3
    ) AS events
    GROUP BY DATE(created_at)
    ORDER BY day ASC
    ```
    Replace the current `heatmapRows` query with this UNION query.
  - **Add `heatmap_label` field to analytics response**: `heatmap_label: 'actions'` (so frontend can say "X actions")
