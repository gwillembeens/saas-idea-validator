# Plan 17-03: Frontend — HistoryCard Badge & Toggle

## Objective

Show a Public/Private badge on each HistoryCard with an inline toggle button. Use Redux for optimistic in-place updates — no full list refetch on toggle.

---

## Tasks

### Task 1 — Add `updateItemVisibility` reducer to `historySlice.js`

**File:** `client/src/store/slices/historySlice.js`

Follow the existing `updateItemTitle` reducer pattern. Find item by id, set `is_public`:

```js
updateItemVisibility: (state, action) => {
  const { id, is_public } = action.payload
  const item = state.items.find(i => i.id === id)
  if (item) {
    item.is_public = is_public
  }
},
```

Export `updateItemVisibility` from the slice actions.

---

### Task 2 — Add `toggleItemVisibility` to `useHistory.js`

**File:** `client/src/hooks/useHistory.js`

Follow the existing `updateItemTitle` function pattern: optimistic dispatch → PATCH → revert on error.

```js
const toggleItemVisibility = useCallback(async (id, currentIsPublic) => {
  const newIsPublic = !currentIsPublic
  dispatch(updateItemVisibility({ id, is_public: newIsPublic }))
  try {
    const res = await fetchWithAuth(`/api/history/${id}/visibility`, {
      method: 'PATCH',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ is_public: newIsPublic }),
    })
    if (!res.ok) {
      dispatch(updateItemVisibility({ id, is_public: currentIsPublic }))
    }
  } catch {
    dispatch(updateItemVisibility({ id, is_public: currentIsPublic }))
  }
}, [dispatch])
```

Return `toggleItemVisibility` from the hook.

### Task 3 — Update `HistoryCard.jsx`: badge + toggle button in footer row

**File:** `client/src/components/history/HistoryCard.jsx`

**New prop:** `onToggleVisibility` callback (receives `id` and current `is_public`)

**Local state:** `isTogglingVisibility` boolean to disable button during in-flight request. Since `useHistory` handles optimistic dispatch synchronously, `isTogglingVisibility` is only needed to prevent double-click — set true on click, set false in `finally`.

**Footer row changes** (lines 70–93): insert Public/Private badge and toggle button between the niche pill and the date.

**Badge style** — match NichePill `size="sm"`: `px-3 py-1 font-body text-xs text-pencil border border-pencil` with bg `#e5e0d8`. No hard shadow (small badge).

**Toggle button style:** Plain text, `font-body text-xs text-pencil opacity-60 hover:opacity-100`. Stop propagation (same as delete button at line 35).

**Icon:** `Globe` (public) or `Lock` (private) from lucide-react, `size={12}`, `strokeWidth={2.5}`.

Example footer insertion:
```jsx
{/* Public/Private badge */}
<span
  className="px-3 py-1 font-body text-xs text-pencil border border-pencil"
  style={{ backgroundColor: '#e5e0d8', borderRadius: '255px 15px 225px 15px / 15px 225px 15px 255px' }}
>
  {item.is_public ? 'Public' : 'Private'}
</span>

{/* Toggle button */}
<button
  onClick={(e) => {
    e.stopPropagation()
    if (!isTogglingVisibility) {
      setIsTogglingVisibility(true)
      onToggleVisibility(item.id, item.is_public).finally(() => setIsTogglingVisibility(false))
    }
  }}
  disabled={isTogglingVisibility}
  className="font-body text-xs text-pencil opacity-60 hover:opacity-100 transition-opacity disabled:opacity-30 flex items-center gap-1"
>
  {item.is_public
    ? <><Lock size={12} strokeWidth={2.5} /> Make Private</>
    : <><Globe size={12} strokeWidth={2.5} /> Make Public</>
  }
</button>
```

Note: `onToggleVisibility` from `useHistory` dispatches optimistically and returns a promise — wrap in `.finally()` to clear local loading state.

## Verification

- Each HistoryCard shows "Public" or "Private" badge matching `item.is_public`
- Clicking toggle immediately flips badge and button label (optimistic)
- Button is disabled while PATCH is in-flight
- If PATCH fails, badge and button revert to original state
- Click on toggle does not navigate to result page (stopPropagation)
- Badge style matches NichePill size="sm" visually
