# Plan 17-02: Frontend — ResultPage Toggle Button

## Objective

Add an owner-only public/private toggle button to the result page via ActionButtons. Optimistic update — flip immediately, revert on error.

---

## Tasks

### Task 1 — Update `ActionButtons.jsx`

**File:** `client/src/components/validator/ActionButtons.jsx`

**New props to accept:** `isPublic`, `onToggleVisibility`, `isTogglingVisibility`

**Icon imports:** Add `Globe` and `Lock` from `lucide-react` (alongside existing imports). Use `strokeWidth={2.5}` per design system.

**Button placement:** After social share buttons, before the delete button. Render only when `result?.isOwner` is true (same gate as delete).

**Button content:**
- When `isPublic === true`: show `<Lock strokeWidth={2.5} />` icon + label `"Make Private"`
- When `isPublic === false`: show `<Globe strokeWidth={2.5} />` icon + label `"Make Public"`

**Button style:** Plain text button matching existing secondary style. Disable while `isTogglingVisibility` is true to prevent double-click race.

Example structure for the toggle button (inside the owner-gated block):
```jsx
<button
  onClick={onToggleVisibility}
  disabled={isTogglingVisibility}
  className="flex items-center gap-2 font-body text-pencil opacity-70 hover:opacity-100 transition-opacity disabled:opacity-40"
>
  {isPublic
    ? <Lock size={16} strokeWidth={2.5} />
    : <Globe size={16} strokeWidth={2.5} />
  }
  <span>{isPublic ? 'Make Private' : 'Make Public'}</span>
</button>
```

### Task 2 — Update `ResultPage.jsx`

**File:** `client/src/pages/ResultPage.jsx`

**Add state:**
```js
const [isTogglingVisibility, setIsTogglingVisibility] = useState(false)
```

**Add handler** (after `handleDelete`):
```js
const handleToggleVisibility = async () => {
  if (!result) return
  const originalIsPublic = result.is_public
  setIsTogglingVisibility(true)
  try {
    const res = await fetchWithAuth(`/api/history/${id}/visibility`, {
      method: 'PATCH',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ is_public: !originalIsPublic }),
    })
    if (res.ok) {
      await refetch()
    } else {
      await refetch()
    }
  } catch (err) {
    console.error(err)
    await refetch()
  } finally {
    setIsTogglingVisibility(false)
  }
}
```

Note: `refetch()` is called on both success and error — this keeps the UI in sync with server truth rather than doing a client-side optimistic mutation on `result` (which is owned by `useHistoryResult` and is immutable from the page's perspective).

**Update `<ActionButtons>` call** (line 174):
```jsx
<ActionButtons
  result={result}
  isDeleting={isDeleting}
  onRevalidate={handleRevalidate}
  onDelete={() => setShowDeleteConfirm(true)}
  isPublic={result?.is_public}
  onToggleVisibility={handleToggleVisibility}
  isTogglingVisibility={isTogglingVisibility}
/>
```

---

## Verification

- Owner sees toggle button labeled "Make Private" when `is_public` is `true`
- Owner sees toggle button labeled "Make Public" when `is_public` is `false`
- Clicking toggle fires `PATCH /api/history/:id/visibility`
- Button is disabled while request is in-flight (prevents double-click)
- Non-owner sees no toggle button
- On 500 error, UI reverts to original state after `refetch()`
