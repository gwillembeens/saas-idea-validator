# Plan 02-02: Express Server Integration & Verification

## Objective

Update `server/index.js` to mount the `validateRoute` handler at `POST /api/validate`. Verify the server starts correctly on port 3001, returns 400 for invalid input, and streams a response for valid input.

## must_haves

- `server/index.js` imports `validateRoute` from `./routes/validate.js`
- `POST /api/validate` route is registered in Express
- Server reads `PORT` from `process.env` (already in scaffold — confirm it's correct)
- `curl` test: short idea → 400 `{ "error": "Idea too short." }`
- Server starts without errors: `node index.js` outputs `Server listening on port 3001`

Read the current `server/index.js` content first, then update it to add the route import and mount. The final `server/index.js` must contain:

```js
import 'dotenv/config'
import express from 'express'
import cors from 'cors'
import { validateRoute } from './routes/validate.js'

const app = express()
const PORT = process.env.PORT || 3001

app.use(cors())
app.use(express.json())

app.get('/health', (req, res) => {
  res.json({ status: 'ok' })
})

app.post('/api/validate', validateRoute)

app.listen(PORT, () => {
  console.log(`Server listening on port ${PORT}`)
})
```

Key changes from current scaffold:

2. Add `app.post('/api/validate', validateRoute)` before `app.listen()`

<task id="2-02-02">
<title>Verify server starts and validate endpoint behavior</title>
<read_first>
- server/index.js — confirm route is mounted (task 2-02-01 must complete first)
- server/.env — confirm ANTHROPIC_API_KEY is set (needed for live streaming test)
</read_first>
<action>
Run verification checks. Requires `server/.env` to have a valid `ANTHROPIC_API_KEY`.

**Step 1: Start server in background**
```bash
cd server && node index.js &
SERVER_PID=$!
sleep 2
```

**Step 2: Health check**
```bash
curl -s http://localhost:3001/health
# Expected: {"status":"ok"}
```

**Step 3: Test 400 on short input**
```bash
curl -s -X POST http://localhost:3001/api/validate \
  -H "Content-Type: application/json" \
  -d '{"idea":"short"}'
# Expected: {"error":"Idea too short."}
```

**Step 4: Test streaming response**
```bash
curl -s -X POST http://localhost:3001/api/validate \
  -H "Content-Type: application/json" \
  -d '{"idea":"A SaaS platform that automates invoice collection and payment reminders for freelance designers using AI agents"}' \
  --no-buffer \
  --max-time 30
# Expected: streaming markdown text arrives progressively, includes "Idea Summary" and "Scorecard"
```

**Step 5: Kill server**
```bash
kill $SERVER_PID 2>/dev/null
```

If `ANTHROPIC_API_KEY` is not set, skip Step 4 and verify Steps 2 and 3 only.
</action>
<acceptance_criteria>
- `curl -s http://localhost:3001/health` returns `{"status":"ok"}`
- `curl -s -X POST http://localhost:3001/api/validate -H "Content-Type: application/json" -d '{"idea":"x"}'` returns JSON with `"error"` key
- Server process starts without Node.js import/syntax errors (check stderr)
- (If API key present) Streaming response contains markdown text chunks (non-empty body)
</acceptance_criteria>
</task>
