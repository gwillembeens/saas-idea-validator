Replace the `status === 'done'` branch in `ResultsPanel` with the following split-card
layout. The existing behavior (single card markdown dump) becomes the fallback when
`parseSections` returns null.

```jsx
if (status === 'done') {
  const sections = parseSections(result)

  if (!sections) {
    // Fallback: parse failed — render raw markdown in single card (existing behavior)
    return (
      <div className="w-full max-w-2xl mx-auto">
        <Card decoration="none" rotate={0} className="w-full">
          <ReactMarkdown components={markdownComponents}>{result}</ReactMarkdown>
        </Card>
      </div>
    )
  }

  return (
    <div className="w-full flex flex-col items-center gap-8 animate-fadeIn">
      <VerdictBadge />
      <IdeaSummaryCard markdown={sections.ideaSummary} />
      <Scorecard />
      <VerdictCard markdown={sections.verdict} />
      <CommentaryCard markdown={sections.commentary} />
    </div>
  )
}
```

Key implementation notes:
- `animate-fadeIn` on the wrapper div (requires Wave 2 Tailwind task)
- `VerdictBadge` and `Scorecard` are Redux-connected components — they take NO props and read state via `useSelector` internally. Render as `<VerdictBadge />` and `<Scorecard />`.
- `IdeaSummaryCard`, `VerdictCard`, `CommentaryCard` receive the parsed section string as `markdown` prop
- Fallback branch renders raw `result` string in a Card (same as existing done state)

Check the current `VerdictBadge` and `Scorecard` prop signatures from their source files
and adjust the prop passed if they differ from `result`. Common alternatives: `markdown`,
`content`. Match what they actually expect.
