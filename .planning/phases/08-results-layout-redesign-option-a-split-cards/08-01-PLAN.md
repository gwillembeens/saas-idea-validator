# Plan 08-01: Parsing Utility & Card Components (Wave 1)

## Overview

Create the `parseSections` utility to split Claude's markdown response by section headings, then build three new card components (`IdeaSummaryCard`, `VerdictCard`, `CommentaryCard`) that render individual sections. These are stateless, reusable components that receive markdown content as props.

## Architecture

**New files:**
- `client/src/utils/parseSections.js` — Split markdown by `## ` headings, return `{ ideaSummary, commentary, verdict }` or `null`
- `client/src/components/validator/IdeaSummaryCard.jsx` — Render Idea Summary section in a Card
- `client/src/components/validator/VerdictCard.jsx` — Render Verdict section in a Card
- `client/src/components/validator/CommentaryCard.jsx` — Render Commentary section in a Card

**Design constraints:**
- All cards use the existing `Card` component with wobbly borders, hard shadows, and design system colors
- Markdown content rendered via the same `markdownComponents` pattern as ResultsPanel
- No Redux dependency — components receive markdown strings as props only
- Graceful fallback: if parsing fails (parseSections returns null), ResultsPanel handles the error

Create `client/src/utils/parseSections.js` with the following implementation:

```javascript
// Splits Claude's markdown response into logical sections.
// Returns { ideaSummary, commentary, verdict } or null on failure.
// Claude's system prompt guarantees these headings in this order:
//   ## 📋 Idea Summary
//   ## 🔬 Scorecard      ← stripped/ignored
//   ## 📝 Commentary
//   ## ✅ Verdict

export function parseSections(markdown) {
  try {
    if (!markdown || !markdown.trim()) return null

    // Split by any ## heading
    const sections = {}

    // Extract Idea Summary: content between ## 📋 Idea Summary and next ##
    const ideaSummaryMatch = markdown.match(/## 📋 Idea Summary\n([\s\S]*?)(?=\n## |\n---\n##|$)/)
    sections.ideaSummary = ideaSummaryMatch ? ideaSummaryMatch[1].trim() : ''

    // Extract Commentary: content between ## 📝 Commentary and next ##
    const commentaryMatch = markdown.match(/## 📝 Commentary\n([\s\S]*?)(?=\n## |\n---\n##|$)/)
    sections.commentary = commentaryMatch ? commentaryMatch[1].trim() : ''

    // Extract Verdict: content between ## ✅ Verdict and end
    const verdictMatch = markdown.match(/## ✅ Verdict\n([\s\S]*?)(?=\n## |\n---\n##|$)/)
    sections.verdict = verdictMatch ? verdictMatch[1].trim() : ''

    // If all sections are empty, parsing completely failed — return null
    if (!sections.ideaSummary && !sections.commentary && !sections.verdict) {
      return null
    }

    return sections
  } catch {
    return null
  }
}
```

Key behaviors:
- Returns null if markdown is falsy (null, undefined, empty string)
- Returns null if all three sections extract as empty (parse failure)
- Returns object with all three keys as strings (empty string if section missing)
- Wraps in try-catch, returns null on any regex error
- Ignores `## 🔬 Scorecard` section entirely (not captured)
- Handles `---` dividers between sections (common in Claude's output format)

  )
}
