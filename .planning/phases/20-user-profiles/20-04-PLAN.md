# Plan 20-04 — Frontend Wave 2: NavBar Updates & Route Registration

## Goal
Add the "Settings" link and 28px Avatar badge to the NavBar for logged-in users, and register the `/profile/:username` and `/settings` routes in `App.jsx`.

## Requirements Covered
- PROF-01: `/profile/:username` route accessible from leaderboard and profile links
- PROF-02: `/settings` route accessible from NavBar
- PROF-05: 28px Avatar in NavBar shows logged-in user's initials

---

## Tasks

### Task 1: Update `client/src/components/layout/NavBar.jsx`

**File:** `client/src/components/layout/NavBar.jsx`
**Action:** Two changes:

**1a: Add Avatar import** at the top of the file after existing imports:

Replace:
```js
import { Link, useLocation } from 'react-router-dom'
import { useDispatch } from 'react-redux'
import { useAuth } from '../../hooks/useAuth'
import { Button } from '../ui/Button'
import { SearchBar } from './SearchBar'
import { reset } from '../../store/slices/validatorSlice'
```

With:
```js
import { Link, useLocation } from 'react-router-dom'
import { useDispatch } from 'react-redux'
import { useAuth } from '../../hooks/useAuth'
import { Button } from '../ui/Button'
import { SearchBar } from './SearchBar'
import { Avatar } from '../ui/Avatar'
import { reset } from '../../store/slices/validatorSlice'
```

**1b: Update the logged-in nav section** to add Settings link + Avatar badge:

Replace:
```jsx
          {user ? (
            <>
              <Link
                to="/history"
                className="font-body text-lg text-blue hover:text-accent transition-colors"
              >
                History
              </Link>
              <Button variant="secondary" onClick={logout}>
                Sign Out
              </Button>
            </>
          ) : (
            <Button variant="secondary" onClick={() => openModal('login')}>
              Sign In
            </Button>
          )}
```

With:
```jsx
          {user ? (
            <>
              <Link
                to="/history"
                className="font-body text-lg text-blue hover:text-accent transition-colors"
              >
                History
              </Link>
              <Link
                to="/settings"
                className="font-body text-lg text-blue hover:text-accent transition-colors"
              >
                Settings
              </Link>
              <Button variant="secondary" onClick={logout}>
                Sign Out
              </Button>
              {user.username && (
                <Link to={`/profile/${user.username}`} aria-label="Your profile">
                  <Avatar
                    displayName={user.displayName}
                    username={user.username}
                    size="sm"
                  />
                </Link>
              )}
            </>
          ) : (
            <Button variant="secondary" onClick={() => openModal('login')}>
              Sign In
            </Button>
          )}
```

**Expected:** NavBar shows `History | Settings | Sign Out | [Avatar]` for logged-in users. Avatar is 28px, links to `/profile/:username`. Avatar only shown if user has a username set. Settings link appears for all logged-in users.

### Task 2: Update `client/src/App.jsx` — add new routes

**File:** `client/src/App.jsx`
**Action:** Two changes:

**2a: Add imports** for the new pages:

Replace:
```js
import { HomePage } from './pages/HomePage'
import { HistoryPage } from './pages/HistoryPage'
import { ResultPage } from './pages/ResultPage'
import { FrameworkPage } from './pages/FrameworkPage'
import { LeaderboardPage } from './pages/LeaderboardPage'
```

With:
```js
import { HomePage } from './pages/HomePage'
import { HistoryPage } from './pages/HistoryPage'
import { ResultPage } from './pages/ResultPage'
import { FrameworkPage } from './pages/FrameworkPage'
import { LeaderboardPage } from './pages/LeaderboardPage'
import { ProfilePage } from './pages/ProfilePage'
import { SettingsPage } from './pages/SettingsPage'
```

**2b: Add new routes** in the Routes block:

Replace:
```jsx
    <Routes>
      <Route path="/" element={<HomePage />} />
      <Route path="/history" element={<HistoryPage />} />
      <Route path="/history/:id" element={<ResultPage />} />
      <Route path="/framework" element={<FrameworkPage />} />
      <Route path="/leaderboard" element={<LeaderboardPage />} />
    </Routes>
```

With:
```jsx
    <Routes>
      <Route path="/" element={<HomePage />} />
      <Route path="/history" element={<HistoryPage />} />
      <Route path="/history/:id" element={<ResultPage />} />
      <Route path="/framework" element={<FrameworkPage />} />
      <Route path="/leaderboard" element={<LeaderboardPage />} />
      <Route path="/profile/:username" element={<ProfilePage />} />
      <Route path="/settings" element={<SettingsPage />} />
    </Routes>
```

**Expected:** Navigating to `/profile/testuser` renders `ProfilePage`. Navigating to `/settings` renders `SettingsPage`. Both integrate with NavBar.

## Verification
- Logged-in NavBar shows: `Leaderboard | Framework | History | Settings | Sign Out` + Avatar (if username set)
- Clicking Avatar in NavBar navigates to `/profile/:username`
- Clicking "Settings" in NavBar navigates to `/settings`
- `/profile/:username` route renders ProfilePage
- `/settings` route renders SettingsPage
- Logged-out NavBar unchanged (no Settings link, no Avatar)
- Avatar not rendered in NavBar when user has no username yet
