# Plan 15-01: Password Reset Frontend Wiring

## Objective

Wire the frontend password reset flow by adding `resetPassword()` to the useAuth hook, implementing the 'reset' mode UI in AuthModal, and confirming App.jsx correctly manages the reset token from sessionStorage. Users will be able to receive a reset email, click the link, submit a new password, and log in with the updated credentials.

## must_haves

- User can submit a new password via a form that appears after clicking the reset link
- `resetPassword()` POSTs to `/api/auth/reset-password` with `{ token, password }`
- sessionStorage reset token is cleared after successful submission
- Success message shown; user redirected to login mode after 2 seconds
- Backend integration works: login with new password succeeds

<task id="15-01-01" title="Add resetPassword function to useAuth hook">
  <read_first>
    - client/src/hooks/useAuth.js
    - server/routes/auth.js (resetPasswordRoute: expects POST body { token, password })
    - client/src/store/slices/authSlice.js (setAuthLoading, setAuthError actions)
  </read_first>
  <action>
    Add the following async function to useAuth.js after the forgotPassword function:

    ```javascript
    async function resetPassword(token, newPassword) {
      dispatch(setAuthLoading())
      try {
        const res = await fetch('/api/auth/reset-password', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({ token, password: newPassword }),
        })
        const data = await res.json()
        if (!res.ok) throw new Error(data.error || 'Password reset failed')
        sessionStorage.removeItem('resetToken')
        dispatch(setAuthError(null))
        return { success: true, message: data.message || 'Password reset successful' }
      } catch (e) {
        dispatch(setAuthError(e.message))
        return { success: false, message: e.message }
      }
    }
    ```

    Add `resetPassword` to the hook's return object alongside login, register, logout, forgotPassword.
  </action>
  <acceptance_criteria>
    - useAuth.js contains `async function resetPassword(token, newPassword)`
    - useAuth.js contains `body: JSON.stringify({ token, password: newPassword })`
    - useAuth.js contains `/api/auth/reset-password`
    - useAuth.js contains `sessionStorage.removeItem('resetToken')`
    - useAuth.js contains `dispatch(setAuthError(null))` on success path
    - resetPassword appears in the return object of the hook
  </acceptance_criteria>
</task>
