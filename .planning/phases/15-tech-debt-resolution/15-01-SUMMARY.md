## Integration Points

### Frontend flow:
1. User clicks reset link in email: `http://localhost:5173/?reset=TOKEN`
2. App.jsx detects `reset` param, stores token in sessionStorage, opens modal in reset mode
3. User sees "Set New Password" form with two password fields
4. User enters and confirms new password
5. AuthModal validates match, calls `resetPassword(token, password)`
6. useAuth sends POST to `/api/auth/reset-password`
7. Backend updates password hash, marks token as used, invalidates refresh tokens
8. Frontend shows success message, redirects to login after 2 seconds
9. User logs in with new password

### Data flow:
- Reset link generated by backend: `forgotPasswordRoute` sends email with `?reset=TOKEN`
- Token stored in DB: `password_reset_tokens` table with expiration and used flag
- Frontend retrieves from sessionStorage, sends to backend via `resetPasswordRoute`
- Backend validates token, updates user password, clears sessions
- Frontend clears sessionStorage and shows login modal
