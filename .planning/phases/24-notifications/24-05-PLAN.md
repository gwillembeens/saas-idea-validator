4. Add a `useEffect` hook to auto-open CommentModal when navigating with state. Add it after the existing useEffect hooks, around line 46:
```javascript
  useEffect(() => {
    if (location.state?.openComments) {
      setShowComments(true)
    }
  }, []) // Only on initial mount
```

5. Verify that the CommentModal is mounted in the JSX somewhere in the return statement. Search for `CommentModal` in the render output. It should look like:
```jsx
      {showComments && (
        <CommentModal
          resultId={id}
          onClose={() => setShowComments(false)}
        />
      )}
```

If CommentModal is not rendered, add it before the closing `</AppShell>` tag:
```jsx
      {showComments && (
        <CommentModal
          resultId={id}
          onClose={() => setShowComments(false)}
        />
      )}
```

The `useEffect` with empty dependency array ensures the auto-open only happens on initial page load, not on re-renders. The `location.state?.openComments` check means it only opens if the navigation explicitly set that state (which NotificationDropdown and NotificationsPage do for comment notifications).
