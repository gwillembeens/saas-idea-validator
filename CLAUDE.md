## Project Overview

A React web app that lets founders paste a startup idea and receive a structured
validation report scored against a 30-step framework for building agent-native SaaS
businesses. The Claude API powers the analysis. The UI uses a hand-drawn / sketchbook
design system (defined below).

## Architecture Rules

1. **API key lives server-side only.** Express reads `process.env.ANTHROPIC_API_KEY`.
   The client never sees it. Vite proxies `/api` to `localhost:3001`.

2. **Redux for all app state.** The `validatorSlice` manages:
   - `idea: string` — textarea content
   - `status: 'idle' | 'loading' | 'streaming' | 'done' | 'error'`
   - `result: string` — accumulated markdown from the stream
   - `error: string | null`

3. **Streaming first.** Express uses `res.setHeader('Content-Type', 'text/event-stream')`.
   The client reads chunks via `ReadableStream` and dispatches `appendResult` on each
   chunk. The UI renders progressively — never wait for the full response.

4. **System prompt is a constant**, defined in `client/src/utils/systemPrompt.js`.
   It is the complete validator framework (included verbatim below).
   Never hardcode it elsewhere.

5. **Components are dumb by default.** Only `IdeaInput` and `ResultsPanel` connect
   to Redux. All UI primitives (Button, Card, TextArea) are pure props-based components.

6. **parseResult.js** scans the streamed markdown for the scorecard table and extracts
   scores to power the visual `Scorecard` component. It should be defensive — if parsing
   fails, fall back to rendering raw markdown only.

7. **One route only.** `POST /api/validate` accepts `{ idea: string }` and streams back
   the Claude response. No auth, no database, no sessions needed for v1.

## Design System — Hand-Drawn Sketchbook

### Philosophy
Reject clinical precision. Embrace organic, playful irregularity. Every element should
feel like it was sketched on paper, not generated by a computer.

Core principles:
- **No straight lines** — wobbly border-radius everywhere (use inline style, not Tailwind)
- **Hard offset shadows** — solid offset, zero blur: `4px 4px 0px 0px #2d2d2d`
- **Handwritten fonts** — Kalam for headings, Patrick Hand for body/inputs
- **Playful rotation** — elements tilted -2deg to 2deg
- **Paper texture** — dot grid background on body
- **Limited palette** — paper white, pencil black, red marker, post-it yellow, blue pen

### Button Component API
```jsx
<Button variant="primary" | "secondary" onClick={fn}>Label</Button>
```
- Primary: white bg → red fill on hover, hard shadow presses flat on active
- Secondary: muted bg → blue fill on hover
- Font: Patrick Hand, minimum h-12 for touch targets

### Card Component API
```jsx
<Card decoration="tape" | "tack" | "none" rotate={1 | -1 | 0}>
  {children}
</Card>
```
- tape: translucent gray bar at top, slight rotation
- tack: red circle at top center

### TextArea Component
- Full wobbly border, no underline-only style
- Focus: border switches to blue `#2d5da1`, ring-2 in blue/20
- Font: Patrick Hand
- No resize handle (resize-none)

### Score Bar (ScoreBar)
Visual 1–5 rating. Use 5 circles. Filled = pencil black `#2d2d2d`. Empty = muted.
Apply hard offset shadow to the filled ones. Slight rotation on the row (-1deg).

### Verdict Badge (VerdictBadge)
| Score | Color | Label |
|---|---|---|
| 4.5–5.0 | green bg | 🟢 Strong Signal |
| 3.5–4.4 | yellow bg | 🟡 Promising |
| 2.5–3.4 | orange bg | 🟠 Needs Work |
| 1.0–2.4 | red bg | 🔴 Too Vague |

Use postit yellow background with pencil border + hard shadow. Rotate -1deg.

### Typography Rules
- All headings: `font-heading` (Kalam 700)
- All body text, labels, inputs, buttons: `font-body` (Patrick Hand 400)
- Hero heading: `text-5xl md:text-6xl`
- Section headings: `text-3xl md:text-4xl`
- Body: `text-lg md:text-xl`
- Buttons: `text-lg md:text-2xl`
- Color: always `text-pencil` (#2d2d2d), never pure black

### Animations
- Hover on cards: `hover:rotate-1` or `hover:-rotate-1`, `transition-transform duration-100`
- Buttons: translate on hover (`translate-x-[2px] translate-y-[2px]`), shadow reduces
- Decorative elements: gentle bounce animation (3s, infinite)

### Responsive Rules
- Mobile-first. All grids collapse to 1 col, expand at `md:` breakpoint
- Hide decorative SVGs on mobile: `hidden md:block`
- Keep wobbly borders and handwritten fonts at ALL screen sizes
- Section padding: `py-20` always

## Express Server (`server/routes/validate.js`)

```js
// POST /api/validate
// Body: { idea: string }
// Streams Claude response as text/event-stream

import Anthropic from '@anthropic-ai/sdk'
import { SYSTEM_PROMPT } from '../systemPrompt.js'  // same prompt, server copy

export async function validateRoute(req, res) {
  const { idea } = req.body
  if (!idea || idea.trim().length < 20) {
    return res.status(400).json({ error: 'Idea too short.' })
  }

  res.setHeader('Content-Type', 'text/event-stream')
  res.setHeader('Cache-Control', 'no-cache')
  res.setHeader('Connection', 'keep-alive')

  const client = new Anthropic()
  const stream = client.messages.stream({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 2000,
    system: SYSTEM_PROMPT,
    messages: [{ role: 'user', content: idea }],
  })

  for await (const chunk of stream) {
    if (chunk.type === 'content_block_delta' && chunk.delta.type === 'text_delta') {
      res.write(chunk.delta.text)
    }
  }

  res.end()
}
```

## Redux Slice (`client/src/store/slices/validatorSlice.js`)

```js
import { createSlice } from '@reduxjs/toolkit'

const validatorSlice = createSlice({
  name: 'validator',
  initialState: {
    idea: '',
    status: 'idle',   // idle | loading | streaming | done | error
    result: '',
    error: null,
  },
  reducers: {
    setIdea: (state, action) => { state.idea = action.payload },
    startValidation: (state) => { state.status = 'loading'; state.result = ''; state.error = null },
    startStreaming: (state) => { state.status = 'streaming' },
    appendResult: (state, action) => { state.result += action.payload },
    finishValidation: (state) => { state.status = 'done' },
    setError: (state, action) => { state.status = 'error'; state.error = action.payload },
    reset: (state) => { state.status = 'idle'; state.result = ''; state.error = null },
  },
})

export const { setIdea, startValidation, startStreaming, appendResult, finishValidation, setError, reset } = validatorSlice.actions
export default validatorSlice.reducer
```

## Streaming Hook (`client/src/hooks/useValidate.js`)

```js
import { useDispatch, useSelector } from 'react-redux'
import { startValidation, startStreaming, appendResult, finishValidation, setError } from '../store/slices/validatorSlice'

export function useValidate() {
  const dispatch = useDispatch()
  const { idea, status, result, error } = useSelector(s => s.validator)

  async function validate() {
    dispatch(startValidation())
    try {
      const res = await fetch('/api/validate', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ idea }),
      })
      if (!res.ok) throw new Error('Server error')
      dispatch(startStreaming())
      const reader = res.body.getReader()
      const decoder = new TextDecoder()
      while (true) {
        const { done, value } = await reader.read()
        if (done) break
        dispatch(appendResult(decoder.decode(value)))
      }
      dispatch(finishValidation())
    } catch (e) {
      dispatch(setError(e.message))
    }
  }

  return { idea, status, result, error, validate }
}
```

## parseResult.js utility

Parses the streamed markdown to extract numeric scores for the visual Scorecard component.

```js
// Returns { phase1: 4, phase2: 3, phase3: 4, phase4: 3, weighted: 3.7 } or null on failure
export function parseScores(markdown) {
  try {
    const rows = markdown.match(/\|\s*([\d.]+)\/5\s*\|/g)
    if (!rows || rows.length < 4) return null
    const scores = rows.slice(0, 4).map(r => parseFloat(r.match(/[\d.]+/)[0]))
    const weighted = +(scores[0] * 0.30 + scores[1] * 0.25 + scores[2] * 0.35 + scores[3] * 0.10).toFixed(1)
    return { phase1: scores[0], phase2: scores[1], phase3: scores[2], phase4: scores[3], weighted }
  } catch {
    return null
  }
}
```

## What NOT To Do

- Never use `create-react-app`
- Never put the API key in the client
- Never use soft/blurred box shadows — hard offset only
- Never use standard `rounded-*` Tailwind classes on primary elements
- Never use system fonts — always Kalam or Patrick Hand
- Never add a database or auth layer in v1
- Never use `useEffect` for data fetching — use the custom hook
- Never let components manage their own Redux state — use the slice
- Never skip streaming — the UX depends on it
